## 本地安装 (推荐)

Prisma CLI 通常安装为**开发依赖项**，这就是为什么在下面的命令中使用了 `--save-dev` (npm) 和 `--dev` (Yarn) 选项。

### npm

用[npm](https://www.npmjs.com/)安装:

```
npm install prisma --save-dev
```

这会将`prisma` 添加到`package.json`中的`devDependencies`里。然后，您可以使用前缀为[`npx`](https://github.com/npm/npx#readme)的 **`prisma`** 命令调用本地安装的 CLI:

```
npx prisma
```

下面是调用 `generate` 命令的示例:

```
npx prisma generate
```

### Yarn (1.19.2+)

用[Yarn](https://yarnpkg.com/)安装:

```
yarn add prisma --dev
```

这会将`prisma` 添加到`package.json`中的`devDependencies`里。然后，您可以使用 **`prisma`** 命令以 `yarn` 为前缀调用本地安装的 CLI:

```
yarn prisma
```

下面是调用 `generate` 命令的示例:

```
yarn prisma generate
```

<Admonition type="warning">

我们建议您保持 `prisma` 和 `@prisma/client` 包的同步，以避免任何意外的错误或行为。

</Admonition>

## 全局安装 (不推荐)

虽然建议在项目[本地安装](#local-installation-recommended)Prisma CLI，但您也可以在机器上全局安装它。

> **警告**: 如果您的机器上有几个 Prisma 项目，那么全局安装可能会导致这些项目之间的版本冲突。

### npm

用 npm 安装:

```
npm install -g prisma
```

然后您可以用 **`prisma`** 命令调用全局安装的 CLI，如下所示:

```
prisma
```

这里有一个调用 `generate` 命令的例子:

```
prisma generate
```

### Yarn

用 Yarn 安装:

```
yarn global add prisma
```

然后可以使用 **`prisma`** 命令调用全局安装的 CLI，如下所示:

```
prisma
```

下面是一个调用 `generate` 命令的示例：

```
prisma generate
```

## 在 CLI 上使用 HTTP 代理

Prisma CLI 支持[自定义 HTTP 代理](https://github.com/prisma/prisma/issues/506)。当处于公司防火墙之后时，这一点尤其重要。

要激活代理，请提供环境变量`HTTP_PROXY`和`/`或`HTTPS_PROXY`。这与[`npm` CLI 的处理方式非常相似](https://docs.npmjs.com/misc/config#https-proxy)。

可以提供以下环境变量:

- [`HTTP_PROXY`](../../../reference/api-reference/environment-variables-reference/#http_proxy) <span class="api"></span> 或 `http_proxy`: 例如，http 的代理 URL `http://localhost:8080`
- [`HTTPS_PROXY`](../../../reference/api-reference/environment-variables-reference/#https_proxy) <span class="api"></span> 或 `https_proxy`: 例如，https 的代理网址 `https://localhost:8080`

> 要获取本地代理，也可以使用[`proxy`](https://www.npmjs.com/package/proxy)模块。

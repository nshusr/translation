<TopBlock>

要在 Windows 上运行 Microsoft SQL Server，请执行以下操作：

1. 如果您无法访问 Microsoft SQL Server 实例，请下载并设置[SQL Server 2019 Developer](https://www.microsoft.com/en-us/sql-server/sql-server-downloads)。

1. 下载并安装[SQL Server Management Studio](https://docs.microsoft.com/en-us/sql/ssms/download-sql-server-management-studio-ssms?view=sql-server-ver15)。

1. 使用 Windows 身份验证登录 Microsoft SQL Server Management Studio (下拉展开 **Server Name**，点击 **&lt;Browse for more...&gt;** 来找到您的数据库引擎):

![SQL Server Management Studio中的新建查询按钮](../../../../doc-images/connect-sql-server.png)

</TopBlock>

## 启用 TCP/IP

Prisma Client 需要启用 TCP/IP。要启用 TCP/IP，请执行以下操作:

1. 打开 SQL Server 配置管理器。

1. 在左侧面板中, 单击 **SQL Server Network Configuration** > **Protocols for MSSQLSERVER**

1. 右键单击 **TCP/IP**，而后选择 **Enable**.

## 启用 SQL 登录身份验证 (可选)

如果您希望在连接 URL 中使用用户名和密码，而不是集成安全，请[开启混合认证模式](https://docs.microsoft.com/en-us/sql/database-engine/configure-windows/change-server-authentication-mode?view=sql-server-ver15)，如下所示：

1. 在对象资源管理器中右键单击数据库引擎，然后再单击**Properties**。

1. 在服务器属性窗口中，单击左侧列表中的 **Security**，勾选 **SQL Server and Windows Authentication Mode** 选项，然后再点击 **OK**。

1. 在对象资源管理器中右键单击您的数据库引擎，然后再点击 **Restart**。

### 启用 `sa` 登录

要启用默认的`sa` (管理员) SQL Server 登录，请执行以下操作:

1. 在 SQL Server Management Studio 的对象资源管理器中， 展开 **Security** > **Logins**，然后再双击 **sa**。

1. 在 **General** 页面，选择 `sa` 账号的密码 (如果您不想执行某个策略，请取消勾选**Enforce password policy**)。

1. 在 **Status** 页面的 **Settings** > **Login**下，勾选 **Enabled**，然后再点击 **OK**.

现在，您可以在连接 URL 中和登录 SQL Server Management Studio 时使用 `sa` 帐户。

> **请注意**: `sa` 用户具有广泛的权限。 您也可以[用更少的权限创建自己的登录](https://docs.microsoft.com/en-us/sql/relational-databases/security/authentication-access/create-a-login?view=sql-server-ver15)。
